HTTP Status Codes Library

1xx Status Codes - Informational

100 - Continue

The Status Code indicates that the initial part of a request has been received and has not yet been rejected by the server.

101 - Switching Protocols

The Status Code indicates that the protocol of the server has switched.

102 - Processing (Deprecated)

Disclamer: Only use this status code for compatibility purposes!
This status code indicates that the server has received the request and is processing it, but a response will take a longer time.

103 - Early Hints

This Status Code gives an early hint to the user, so the client can preconnet/preload a ressource

2xx Status Codes - Successful

200 - Ok

This Status Code indicates a request succeded. The response can give back Information.

201 - Created

This status Code indicates that a ressource was created. This could be a user, a form Input, etc.

202 - Accepted

The Status Code indicates that the request is being processed but it is not guaranteed the request will be successful. It comes often with a response body to track the progress

203 - Non-Authoritative Information

The Status Code informs the user, that the original request succeded but the response was changed by a proxy like malware-filtering, etc.

204 - No Content

This status code informs the user that their request succeded, but no action on the client side is required. This is used in cases where something was changed or deleted.

205 - Reset Content

This Status code indicates that the request succeded and the client should reset their page. This is useful, when for example the user inputs something into a form.

206 - Partial Content

The Status Code is sent after a range request is made. This includes the partial date, from the range.

207 - Multi Status

This Status Code is returned, when there are multiple Status Codes returned. It is formatted with xml adn has a multistatus root element

208 - Already Reported

This Status Code is used only in the context of WebDAV. It is used with 207 to shorten the response, if one ressource is requested from multiple paths.

226 - IM Used

Note: Has poor support(only a few implementations)
This Status Code is used, if the user has a version of a ressource, that is not the latest version. The server will then return the latest version of the ressource, but only the changes to the ressource, not the whole ressource.

3xx Status Codes - Redirect

300 - Multiple Choises

This Status Code is used to give the user-agent a list of possible requests, like a website in multiple languages. The user agent could choose one option but most agents lack this skill

301 - Moved Permanently

Note: This should only be used with GET-requests, as the method changes to GET.
This Status Code informs the user, that the ressource has a new location. The browser will automaticly follow the response to the new location.

302 - Found

Note: This should only be used with GET-requests, as the method often changes to GET.
This Status Code informs the user, that the ressource they requested has been moved temporarly to a new location. The browser will automaticly follow the response to the new location.

303 - See other

The Status Code informs the user, that the request was successful but to see changes they need to do a GET Request.

304 - Not Modified

The Status Code informs the user, that nothing has changed since the last request, so there is no need to retransmit the ressource.

307 - Temporary Redirect

This Status Code informs the user, that the ressource they requested has been moved temporarly to a new location. The browser will automaticly follow the response to the new location.
Note: Request Body and Method are preserved.

308 - Permanent Redirect

This Status Code informs the user, that the ressource has a new location. The browser will automaticly follow the response to the new location.
Note: Request Body and Method are preserved.
